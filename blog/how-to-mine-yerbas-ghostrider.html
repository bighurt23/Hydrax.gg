How to Mine Yerbas (YERB) and Raptoreum (RTM): GhostRider CPU Guide (2026)

GhostRider is the CPU algorithm behind Yerbas (YERB) and Raptoreum (RTM), two coins you can mine on the processor you already own, no graphics card required. As of v3.4.5, HydraX mines GhostRider with cleanroom code at a flat 1% dev fee on Windows, Linux and HiveOS, and it can run right alongside a GPU coin so nothing on your rig sits idle.

What is GhostRider, and why it runs on your CPU

GhostRider chains fifteen different hash functions together with CryptoNight rounds. That mix is deliberately ASIC and FPGA resistant, which is the whole point: it keeps mining on ordinary CPUs and in the hands of regular people instead of specialized hardware farms. The algorithm leans heavily on memory and cache latency, so a processor with a big cache does more work per clock than raw core count alone would suggest.

Two coins carry GhostRider today: Yerbas (YERB) and Raptoreum (RTM). They share the algorithm, so one miner handles both, you just point it at the right coin, pool and wallet.

What you need before you start

  1. A modern CPU. Any recent x86-64 processor with AES support works. Chips with large L3 cache lead the pack, and AMD's X3D parts are especially strong on GhostRider.
  2. A wallet address. Yerbas addresses start with y; Raptoreum addresses start with R.
  3. A pool. For Yerbas, HydraX ships pointed at stratum.kriptokyng.com. For Raptoreum, pick any Raptoreum pool and drop it in.
  4. The miner. HydraX mines GhostRider on Windows, Linux and HiveOS at a flat 1% dev fee, and it is code-signed by verified publisher HYDRAX LLC.

How to mine Yerbas (YERB), step by step

  1. Download HydraX for your OS from the releases page.
  2. Open the Yerbas launcher (mine-yerbas.bat on Windows, mine-yerbas.sh on Linux and HiveOS) and paste in your y-prefixed Yerbas address.
  3. Pick solo or pool. The launcher is pre-set to stratum.kriptokyng.com:4533 for solo. On a smaller rig, change the port to 5533 for steady pool payouts.
  4. Run it. Double-click the .bat on Windows, or bash mine-yerbas.sh on Linux and HiveOS. That is it, no config files.

Raptoreum is the same flow: open the launcher, swap the pool for a Raptoreum pool, paste your R address, and run. HydraX auto-adapts to whatever pool you point it at.

Tuning GhostRider for more hashrate

Because GhostRider is cache and memory-latency bound, the biggest levers are memory-side:

  • Large / huge pages. These cut memory-access overhead and can add a meaningful boost. HydraX enables them automatically when the OS allows it (on Windows, run it as administrator so it can grab the privilege).
  • Cache is king. A CPU with more L3 cache, and AMD X3D parts in particular, will out-hash a chip with more cores but less cache.
  • Thread count. HydraX auto-detects a sensible thread count. Solo GhostRider uses all your cores; when you tri-mine with a GPU coin it holds two back for the card. You can override it in HydraX.conf if you want to.

Is it profitable?

GhostRider coins are CPU-mined and easy to get into, but like any coin the payout rides on price, network difficulty and your power cost. Run your CPU's hashrate and electricity rate through a current profit calculator before committing, and remember that stacking a GPU coin on the same rig is what really moves the economics.
