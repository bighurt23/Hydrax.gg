How to Mine Xelis (XEL): CPU Mining Guide (2026)

Guide · ~9 min read · by the Hydrax.GG team

Xelis is one of the few coins in 2026 that is genuinely built for CPUs. Its xelishashv3 algorithm is memory-hard by design — so the processor in your desktop can compete, and your GPU stays free to mine something else.

This guide covers what xelishashv3 actually wants from your hardware, the exact steps to point a miner at a XEL pool, the Windows tuning that adds real hashrate, and — bluntly — what the earnings look like at mid-2026 prices, so you go in with your eyes open. If you're brand new to mining in general, start with our beginner's guide to mining crypto first, then come back.

What is xelishashv3, and why it favors CPUs

Xelis mines with xelishashv3, live since December 2025. It's a memory-hard algorithm that chains BLAKE3, ChaCha8 and AES over a 544 KB per-hash scratchpad. That scratchpad is the point: it's sized to live in a CPU's cache hierarchy, and the constant random access to it punishes the wide-but-cache-poor parallelism that makes GPUs and ASICs dominant on other algorithms. The result is a coin that is deliberately CPU-favored and GPU/ASIC-resistant — the same design philosophy as Monero's RandomX, with a different recipe.

The practical upshot: a modern desktop CPU is competitive hardware here, and the chips that shine are the ones with the most fast cache. If you're weighing CPU coins against GPU coins more broadly, our ASIC vs GPU mining comparison explains why algorithm design decides which hardware wins.

What hardware do you need?

Any modern desktop CPU can mine XEL, but cache is king:

  • Large-L3 chips shine. AMD's X3D parts with 96 MB of L3 cache are standout performers, because more of those 544 KB scratchpads stay in cache instead of spilling to slow system RAM.
  • Expect roughly the ~12 kH/s class from a top desktop CPU on v3. One important warning: older 18–19 kH/s figures still circulating online are from the previous v2 algorithm — v3 is a different beast, and those numbers are not comparable. Don't plug v2 hashrates into a profit estimate.
  • SMT helps. This algorithm scales with logical cores, so leave Hyper-Threading / SMT enabled.
  • RAM matters less than cache, but you'll want enough free memory for the miner's scratchpads across all threads, plus large-pages headroom (more on that below).

Step 5: Tune it (Windows tips that actually move the needle)

  • Enable "Lock pages in memory." Granting your user this Windows policy lets the miner use 2 MB large pages, which cuts address-translation overhead on a memory-hard algorithm. It's worth roughly +8% — the single best free tweak available.
  • Use all logical cores for XEL-only mining. SMT helps this algorithm, so a 16-thread chip should run 16 mining threads when the box is dedicated.
  • Leave ~2 cores free when dual-mining. If the same machine's GPU is mining a coin like Ethereum Classic, the GPU's host thread needs CPU time to keep it fed. Pegging every core can starve the GPU and cost far more there than the extra CPU threads earn. Two reserved cores is the sweet spot.
  • Watch temperatures. A CPU pinned at 100% load all day needs a competent cooler and decent airflow. Throttling quietly eats your hashrate.

Frequently asked

What hashrate does a CPU get mining Xelis? On xelishashv3 (live since December 2025), a top desktop CPU lands in roughly the ~12 kH/s class. Ignore the 18–19 kH/s figures still floating around — those are from the old v2 algorithm and aren't comparable. Large-L3 chips like AMD's 96 MB X3D parts do especially well.

Can I mine Xelis with a GPU or an ASIC? Xelis is deliberately CPU-favored: the 544 KB memory-hard scratchpad with BLAKE3 + ChaCha8 + AES resists GPU and ASIC parallelism. Put your GPU on a GPU coin instead and let the CPU handle XEL — that's the dual-mining play.

Is Xelis mining profitable in 2026? Not as a primary earner — at roughly $0.40 per XEL, a high-end CPU grosses well under $1/day. It works as diversification, speculation, or near-free extra revenue alongside GPU mining. Run your exact CPU and power rate through the calculator before committing.

Does SMT or Hyper-Threading help? Yes — use all logical cores when the machine only mines XEL. If you're dual-mining next to a GPU coin, reserve about two cores so the GPU stays fed.
